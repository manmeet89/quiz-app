Application Architecture

  • The twelve-factor methodology can be applied to apps written in any programming language, and which use any combination of backing services (database, queue, memory cache, etc).
    • http://12factor.net/
  • Design Patterns like Circuit breakers, service discovery etc. are well suited for cloud.
  • SpringBoot : Build Jars not War !
    • https://spring.io/guides/gs/spring-boot/
  • SpringCloud : Spring Cloud provides tools for developers to quickly build some of the common patterns in distributed systems (e.g. configuration management, service discovery, circuit breakers).
    • http://projects.spring.io/spring-cloud/
  • Go beyond Java ... Ruby or Go can be handy to create cloudy apps.
    • https://golang.org/
  • Continuous Integration is not an option it is mandatory for Cloud.
  • Spin up / spin down your app as and when required.
  • Resources are not free !!! So prepare to learn charge back mechanism and ingrain that in your code.
    • https://www.python.org/ OR https://www.gnu.org/software/bash/
